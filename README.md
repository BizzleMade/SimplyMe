# 🪼 SimplyMe Theme
A Jellyfin theme Inspired from Abyss by AumGupta, ElegantFin by lscambo13, and Ultrachromic by CTalvio. I simply wanted to take elements I liked the most from their themes and create my own basic but beautiful, simple yet elegant theme.
It was made for me but can also be specifically customized to fit your preferences.

#### **Author:** [BizzleMade](https://github.com/BizzleMade)

<hr>

### 🎨 Color Scheme

- your choice
- color neutral
- not predefined
- Blue Radiance looks best
- Dark Theme is the runner up
- any color scheme works well with it

This means it takes on the color scheme of whatever the user chooses in the display settings or you can write your own code for a specific color palette and then add it to the css file yourself for more color customization.

### 📝 The CSS File
I am new to coding and this is the first code I have written and put together. To help me get a understanding of it I've read through every single line of the css files for Abyss, ElegantFin, and some elements of Ultrachromic.

As for the assets used there are three from Ultrachromic I used directly which is the rounded corners, compact list, and grid list. Everything else was written from scratch, reverse engineered, or stripped down to bare bones. 

My goal was to make the code as simple as possible, minimal, easy to understand, and customizable while still being able to add your own code or remove what's already there without it breaking. I wrote each asset of the code separately,
took the three from Ultrachromic, and combined them to complete the theme. I tested each asset individually to make sure they worked separately from each other and worked seamlessly together. The file's structure is
numbered with each section being easily identifiable with some instructions for ease of use.

### 👨‍🔧 How to install this theme

👇 Paste the below URL in the "Custom CSS code" box:

    @import url("https://cdn.jsdelivr.net/gh/BizzleMade/SimplyMe@main/SimplyMe/SimplyMe-by-BizzleMade.css");

<details>
  <summary><i>Steps for server-side installation</i></summary>

1. Open settings and click on Dashboard right under the Administration section
2. From the side bar select the Branding tab if you are on Jellyfin 10.11.X or the General tab on older version.
3. Scroll down until you see the "Custom CSS code" box.
4. Paste the above URL in the "Custom CSS code" box.
5. Click save
 </details>

<details>
  <summary><i>Steps for client-side installation</i></summary>

1. Open settings and click on the Display tab
2. Scroll down until you see the "Custom CSS code" box.
3. Paste the above URL in the "Custom CSS code" box.
4. Click save.
 </details>

### 🤝 Compatibility

Custom CSS in Jellyfin natively applies to web-based clients. Depending on how third party streaming hardware packages its native backend applications, elements of the theme may not fully adapt. This is a limitation of
client side rendering engine compatibility rather than a theme bug. From my understanding and what I learned of it I will give a breakdown on its compatibility, how it works, and address any future potential bug request.

<div align="center">

#### Breakdown of Device Compatibility
</div>

| Client Type                                           | Compatibility Status                                |
| ----------------------------------------------------- | --------------------------------------------------- |
| Web Browsers (Chrome, Firefox, Safari, Edge, etc)     | Fully Supported                                     |
| Jellyfin Media Player (Desktop)                       | Fully Supported                                     |
| Smart TV Browsers (LG webOS, Samsung Tizen, etc)      | Supported (May vary by firmware)                    |
| Streaming Devices (Roku, Fire Stick, Android TV, etc) | Limited / UI Not Supported (Native app limitations) |

<div align="center">

### Breakdown of Web vs. Native Apps
</div>

Think of Jellyfin as having two different ways it talks to devices on its backend end and how they both operate.
1. **The Web Engine:** When you log into Jellyfin on a web browser, the desktop player, or a smart TV browser, the app is essentially rendering a website. Because it's running a web engine it reads your .css stylesheet like a blueprint
and paints the screen exactly how its designed. Results may vary depending on the device.

2. **The Native Wrapper:** Devices like Roku, Fire Sticks, and Android TV don't use web browsers to show the media center. They use native programming languages (like Java or BrightScript) to draw the buttons, grids, and posters.
Because they aren't using a web engine, they completely ignore custom web stylesheets (.css). It's a completely different backend architecture in use which is why the theme won't show up on them.
